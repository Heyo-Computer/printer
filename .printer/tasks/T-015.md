## Notes

- 2026-04-30T16:05:03Z [sarocu] ## Plan: sandbox creation commands in config

Goal: extend the global config schema (T-014) so users can override or augment the driver's `create`/`enter`/`destroy` templates without forking the plugin manifest. This is the "knob" most people will reach for: change the base image, add a setup step, mount a cache.

### Schema additions

Append to `[sandbox]` in `~/.printer/config.toml`:

```toml
[sandbox.commands]
# Each key, if set, overrides the matching driver template from the plugin
# manifest. Same {var} interpolation rules as the driver block (T-012).
create  = "heyvm worktree create --base {base_image} --name printer-{spec_slug}"
enter   = "heyvm worktree exec {handle} -- {child}"
destroy = "heyvm worktree destroy {handle}"

# Optional preflight script inside the sandbox right after create. Useful for
# `apt-get install`, `cargo fetch`, etc. Runs once per exec.
post_create = "bash -lc 'cargo fetch || true'"
```

### Files / types

- Extend `printer/src/config.rs::SandboxConfig` with `commands: SandboxCommands` (struct of `Option<String>` fields).
- `drivers.rs::ResolvedDriver::resolve` precedence: config override > plugin manifest > error if neither.
- Add new interpolation var `{spec_slug}` (basename of spec, sanitized) and `{base_image}` (from config) to the driver context — implement in `drivers.rs` next to `HookContext::vars`.
- `post_create` runs as one extra `enter`-wrapped shell call right after the create step; failure aborts the run unless `on_failure = "warn"` (reuse `OnFailure` from `hooks.rs:65`).
- Document the merge order and var list in `HOOKS.md` under a new "Sandbox config" section.

### Verification

- Unit tests in `config.rs`/`drivers.rs`: command override beats manifest; absent override falls through; `{spec_slug}` is sanitized (no slashes).
- `cargo test -p printer`.
- Manual: set `commands.create` to `echo override-{spec_slug}` and run `printer exec` with the heyvm plugin installed; the printed driver line should show the override, not the heyvm default.
