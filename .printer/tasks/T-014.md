## Notes

- 2026-04-30T16:04:47Z [sarocu] ## Plan: global sandbox config file

Goal: introduce `~/.printer/config.toml` to store user-level sandbox preferences (which driver, default base image, env passthrough, mounts). Read once per `run`/`review`/`exec` and merged with plugin driver defaults.

### Schema (initial)

```toml
[sandbox]
# "auto" (pick the only installed driver), "off" (disable), or a plugin name.
driver = "auto"
# Forwarded to the driver's create template as {base_image}.
base_image = "heyvm:ubuntu-22.04"
# Extra env vars exposed inside the sandbox.
env = ["GITHUB_TOKEN", "ANTHROPIC_API_KEY"]
# Extra read/write mounts (host:guest), beyond the cwd which is mounted by default.
mounts = []
```

### Files / types

- New module `printer/src/config.rs`:
  - `pub struct GlobalConfig { sandbox: SandboxConfig }`
  - `pub fn load() -> Result<GlobalConfig>` — reads `data_dir().join("config.toml")` (use existing `plugins::store::data_dir` at `plugins/store.rs:42`); missing file → defaults.
  - `serde(default)` everywhere so partial files don't error.
- Wire into driver loading (T-012): `DriverSet::load_installed(&cfg)` consults `cfg.sandbox.driver` to pick among installed drivers and to gate `--no-sandbox` short-circuit.
- `printer/src/cli.rs`: small subcommand `printer config show` (read-only; pretty-print resolved config) and `printer config edit` (open `$EDITOR` on the file, creating it from a template if absent). Keep this scoped — no `set` subcommand yet.
- Template seed file lives at `printer/src/config.rs` as a `const DEFAULT_CONFIG: &str`.
- `printer/src/plugins/store.rs::data_dir` already creates `~/.printer/`; reuse it.

### Verification

- Manual: `printer config show` on a fresh box prints defaults; create the file with `driver = "off"` and confirm `printer exec` skips sandboxing.
