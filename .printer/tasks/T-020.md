Follow-up to T-017's blocking-turn ACP transport. Add: (a) line-by-line streaming of session/update content blocks to printer's stderr while the turn is in flight (matches the verbose heartbeat UX in session.rs); (b) Ctrl-C cancellation routed through session/cancel + child kill (mirrors session::kill_subtree); (c) permission-mode mapping — Claude's permission_mode flag has no direct ACP analogue, so decide whether to pass through via initialize capabilities, set per-prompt, or document as no-op. Touches printer/src/agent/acp.rs and session.rs. Verification: manual run against a real ACP server (claude-code-acp or poolside) with a long prompt; Ctrl-C must terminate cleanly without orphaning the child.

## Notes

- 2026-05-01T18:30:19Z [sarocu] Plan — three independent slices, all in `printer/src/agent/acp.rs` (transport) + `printer/src/session.rs` (driver glue). Pin behavior in `printer/HOOKS.md`'s ACP section.

Slice A — streaming to stderr (parity with `Session::turn`'s heartbeat UX):
- In `AcpClient::prompt_blocking`, when each `session/update` notification arrives for the active sessionId, emit `eprintln!("[agent] {chunk}")` for any `agent_message_chunk` text alongside the existing `collect_update_text` accumulator. Match `agent_thought_chunk` separately as `[agent:thought]` so the user can tell visible output from inner monologue. Tool-call updates (`tool_call`, `tool_call_update`) should print one short line: `[agent:tool] <name> <status>`.
- Drive a heartbeat in `Session::turn_acp` mirroring `heartbeat_loop` from session.rs:257 — share an `agent_active: AtomicBool` set whenever a notification arrives, spawn the same spinner task when `self.verbose`, stop it on completion. Refactor `heartbeat_loop` to be reusable (already takes the shared atomics) — only the spawn site needs to change.
- TTY handling: same `\r\x1b[2K` clear-and-print pattern as the one-shot path so the spinner does not get clobbered.

Slice B — cancellation:
- In `Session::turn_acp`, wrap the `client.prompt_blocking(...)` future in a `tokio::select!` against `tokio::signal::ctrl_c()` (same shape as session.rs:136-163).
- On Ctrl-C: call a new `AcpClient::cancel(session_id)` that fires `session/cancel` (notification, not request — per ACP spec it is fire-and-forget), wait up to ~500ms for the in-flight `session/prompt` to resolve with a `cancelled` stopReason, then `kill_subtree` the child via the existing `kill_subtree` helper (lift it out of session.rs to a shared util, since both code paths need it). The child is already in its own process group from `AcpClient::spawn` (acp.rs:86-87), so the same SIGKILL-to-pgid call works.
- Drop the `#[allow(dead_code)]` on `AcpClient::shutdown` and route it through both Drop-on-rotate and Ctrl-C. Make `shutdown` non-consuming or provide a `cancel(&self)` variant since `Session` holds the client by value across turns.
- On rotate (`Session::rotate`), call `shutdown` before nulling `self.acp` so the previous server's pgid is reaped instead of leaking.

Slice C — permission-mode mapping:
- Today `permission_mode` is a Claude-specific `--permission-mode <mode>` flag (mod.rs:154). ACP's `initialize` carries `clientCapabilities`; per-session permission policy is server-defined. Decision: pass `permission_mode` only as a hint via `session/new`'s params under a non-standard `_printerPermissionMode` key (servers must ignore unknown fields per JSON-RPC convention) AND surface it via `PRINTER_PERMISSION_MODE` env var on the spawned child. Document the default mapping table in HOOKS.md: `bypassPermissions` → no-op (server still applies its own policy), `acceptEdits` → no-op + warning, `default`/`plan` → no-op + warning. This is intentionally conservative: ACP doesn't standardize this, so don't pretend it does.
- Add a one-line `eprintln!` at session bootstrap: `[printer] acp: --permission-mode is advisory; the ACP server enforces its own policy`.
