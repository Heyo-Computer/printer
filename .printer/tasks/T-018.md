## Notes

- 2026-05-01T18:21:23Z [sarocu] Plan — extend the plugin manifest schema so plugins can contribute an ACP agent (analogous to how the heyvm plugin contributes a `[driver]`).

Why a plugin role: ACP servers are vendor-specific (claude-code-acp, poolside, codex). Launch command, env, and capabilities differ per vendor. Today plugins contribute hooks and drivers; this adds a third role: `[[agent]]` (kind = "acp").

Files to touch:
- `printer/src/plugins/store.rs` — extend `Manifest` with `pub agents: Vec<AgentSpec>` (`#[serde(default, skip_serializing_if = "Vec::is_empty")]`). Add `agent` to the `ROLES` column in `list_installed`.
- New module `printer/src/agents.rs` (mirrors `drivers.rs`):
  * `AgentSpec { kind: AgentKind, name: String, command: String, args: Vec<String>, env: BTreeMap<String,String> }` with `AgentKind::Acp` initially.
  * `validate_agent` — non-empty command; reserved names (`claude`, `opencode`) refused so they cannot shadow built-ins.
  * `ResolvedAgent { plugin, plugin_dir, spec }` and `AgentSet::load_installed` — same shape as `DriverSet`.
  * `AgentSet::resolve(name)` — look up by `name` (one plugin may ship multiple ACP servers).
- `printer/src/cli.rs` — extend the agent flag. Today `AgentKind` is Claude/Opencode. Add ACP via `--agent acp:<name>` parsed in CLI layer, surfaced as `AgentKind::Acp { name }`. Update every command (Run/Review/Exec/Plan/SpecFromFollowups). Avoid a second `--acp-agent` flag.
- `printer/src/agent.rs` — when `kind == Acp { name }`, consult `AgentSet` to resolve `command`/`args`/`env`, and hand them to the ACP transport from T-017. The transport stays plugin-agnostic.
- `printer/src/plugins/install.rs` — call `validate_agent` in the install-time validation path, mirroring `validate_driver`.
- `printer/HOOKS.md` — new "ACP agents" section: schema, `kind = "acp"`, name resolution rule, env contract (`PRINTER_AGENT_NAME`, `PRINTER_PLUGIN`).

Schema (proposal, pin in HOOKS.md):

  [[agent]]
  kind = "acp"
  name = "claude-code"
  command = "claude-code-acp"
  args = ["--no-color"]
  env = { ANTHROPIC_API_KEY = "${env:ANTHROPIC_API_KEY}" }

Verification:
- `cargo test -p printer plugins::store` — parse `[[agent]]`, reject reserved names, multi-agent plugins.
- `cargo test -p printer agents::` — `AgentSet::resolve`, install-time validation, error on duplicate name across plugins.
- Manual: install a stub plugin with `[[agent]] name = "stub"`, run `printer plugins` and confirm `agent` appears under ROLES.

- run.rs / review.rs / plan.rs / spec_from_followups.rs call agents::resolve_acp_launch(&args.agent, args.acp_bin, &args.acp_args) before constructing AgentInvocation. Plugin manifest's command is the default; --acp-bin overrides; --acp-arg appends after the manifest's args.
