- 2026-04-30T16:05:24Z [sarocu] ## Plan: bundle a heyvm plugin with [driver] block

Goal: add `plugins/heyvm/` to the monorepo so `printer add-plugin path:plugins/heyvm` (or the registry entry once T-013 lands) installs a working VM driver that uses `heyvm worktree` for sandbox lifecycle. This is the first concrete consumer of the framework from T-012.

### Layout

```
plugins/heyvm/
  printer-plugin.toml      # [driver] + minimal hooks
  README.md                # how to use, prereqs (heyvm CLI on PATH)
  skills/
    heyvm-sandbox/SKILL.md # tells the agent it is inside a worktree, how to inspect it
```

No Cargo crate — heyvm is an external CLI shipped by https://docs.heyo.computer. This is a **skill-only / driver-only plugin** (the install path at `plugins/install.rs:122` already supports a Cargo.toml-less source dir). The driver block alone makes the plugin useful even without a `binary`.

### `printer-plugin.toml` (sketch — final wording depends on heyvm's actual flags)

```toml
assets = ["skills"]

[driver]
kind = "vm"
# `heyvm worktree create` should print the new worktree id on stdout
create  = "heyvm worktree create --base {base_image} --name printer-{spec_slug}"
enter   = "heyvm worktree exec {handle} -- sh -c {child}"
sync_in = "heyvm worktree push {handle} {cwd}"
sync_out = "heyvm worktree pull {handle} {cwd}"
destroy = "heyvm worktree destroy {handle}"

[[hooks]]
type = "agent"
event = "before_run"
skill = "skills/heyvm-sandbox/SKILL.md"
```

Note on enter template: chose 'heyvm worktree exec {handle} -- sh -c {child}'. drivers.rs::shell_quote wraps the agent argv in single quotes, so heyvm sees a single shell-quoted string after '--'; sh -c then re-parses it inside the worktree. If a heyvm release ships an argv-passthrough form, swap to 'heyvm worktree exec {handle} -- {child}'.
