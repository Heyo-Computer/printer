## Notes

- 2026-04-30T16:04:14Z [sarocu] ## Plan: VM driver hook framework

Goal: let a plugin declare itself as the **sandbox/VM driver** for run/review, so printer dispatches the agent inside a sandbox instead of the host cwd. Models the lifecycle as a new plugin role rather than overloading existing CLI hooks.

### Design

- New manifest section `[driver]` (optional, plugin manifest level — sibling of `[[hooks]]`):
  - `kind = "vm"` (only kind for now)
  - `create = "<shell template>"` — provision a sandbox; must print the sandbox handle (id/path) on stdout
  - `enter = "<shell template>"` — wrap a child command, e.g. `heyvm worktree exec {handle} -- {child}`
  - `sync_in = "<shell template>"` (optional) — push working dir into sandbox before run
  - `sync_out = "<shell template>"` (optional) — pull artifacts back after run/review
  - `destroy = "<shell template>"` (optional) — tear down on clean exit
  - Interpolation vars: `{cwd}`, `{spec}`, `{handle}`, `{child}`, `{phase}` plus anything from `HookContext::vars`.
- Only one driver active at a time. If multiple installed plugins declare `[driver]`, error unless global config selects one (T-014).

### Files / types

- `printer/src/hooks.rs` — extend `HookSpec`/`Manifest` parse, OR add a peer module `printer/src/drivers.rs` (preferred to keep hook code clean):
  - `pub struct DriverSpec { kind, create, enter, sync_in, sync_out, destroy }`
  - `pub struct DriverSet { active: Option<ResolvedDriver> }` with `load_installed(global_cfg) -> Result<DriverSet>`
  - `ResolvedDriver` carries the owning plugin name + dir + interpolated launchers
- `printer/src/plugins/store.rs::Manifest` — add `#[serde(default)] driver: Option<DriverSpec>`
- `printer/src/plugins/source.rs::SourceManifest` — accept `[driver]`, validate at install (round-trip).
- `printer/src/agent.rs` — add an optional `command_wrapper: Option<&ResolvedDriver>` to `AgentInvocation`; when present, the spawned shell command is wrapped via `enter` template.
- `printer/src/run.rs` / `review.rs` — call `DriverSet::load_installed`. If active driver:
  1. invoke `create` once, capture handle
  2. run optional `sync_in`
  3. pass driver into `Session::new`/`AgentInvocation` so each turn execs through `enter`
  4. on completion run `sync_out` then `destroy` (RAII guard analogous to `_watch_guard`)
- `printer/src/exec.rs` — share one driver session across run+review (mirror codegraph_watch guard pattern at `exec.rs:295`).
- `printer/src/cli.rs` — add `--no-sandbox` to `RunArgs`/`ReviewArgs`/`ExecArgs` to bypass driver; matches existing `--no-codegraph-watch` shape.
- `printer/src/main.rs` + `printer/HOOKS.md` — document the new `[driver]` block.
